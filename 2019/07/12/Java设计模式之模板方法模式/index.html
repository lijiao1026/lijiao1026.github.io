Java设计模式之模板方法模式

发表于 2019-07-12 | 阅读次数:

概念

策略模式属于行为模式中的一种，定义一个操作中的算法的框架，而将一些步骤延迟到子类中，使子类可以不改变一个算法的结构。可以重新定义该算法的某些特定步骤。

原理

主要原理在于不修改主类的模型方法，而是修改子类中的实际方法，从而得到不同的结果。

解决问题

  1. 提高代码复用性，将相同部分的代码放在抽象的父类中，而将不同的代码放入不同的子类中
  2. 实现了反向控制，通过一个父类调用其子类的操作，通过对子类的具体实现扩展不同的行为，实现了反向控制 & 符合“开闭原则”

实例

  1. 比如我们吃午饭，这里面有一套操作，比如下楼，去食堂，选择饭菜，然后吃完。我们可以写一个抽象类定义这一组方法。

  2. 比如我要吃面，那我在食堂可以去吃面的窗口。

  3. 比如我要吃叉烧饭，那我在食堂就可以去吃叉烧饭的窗口。
